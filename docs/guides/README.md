## Contributing Guides

Want to add a guide? Great! Here's what makes a good guide:

- **Clear objective**: What will users accomplish?
- **Prerequisites**: What do users need to know first?
- **Step-by-step**: Break down complex tasks
- **Code examples**: Show, don't just tell
- **Screenshots**: Visual aids when helpful
- **Common issues**: Anticipate problems
- **Next steps**: What to explore next
